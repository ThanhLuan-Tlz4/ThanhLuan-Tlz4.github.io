### Week 8 Objectives:

* Understand the Shared Responsibility Model on AWS.

* Practice using AWS Security Hub to monitor security posture.
* Learn how to optimize Amazon EC2 costs using AWS Lambda.
* Practice managing AWS resources using Tags and Resource Groups.

### Week 8 Achievements:

* Understood the **Shared Responsibility Model**, thereby clearly distinguishing the responsibilities of AWS and customers in protecting infrastructure, operating systems, data, and the services used on the Cloud platform.

* Understood the functions of **AWS Identity and Access Management (IAM)**, **Amazon Cognito**, **AWS Organizations**, **IAM Identity Center (SSO)**, and **AWS Key Management Service (KMS)**, while understanding the role of each service in identity management, user access control, and data encryption.

* Successfully deployed **AWS Security Hub**, enabled security assessment standards, collected Security Findings, and monitored the security posture of the AWS account through a centralized dashboard.

* Practiced analyzing security alerts aggregated by Security Hub from multiple AWS services, thereby understanding how to detect insecure configurations and review recommendations for improving system security.

* Successfully built an Amazon EC2 cost optimization solution using **AWS Lambda**, configured an IAM Role and EventBridge to automatically start or stop EC2 Instances according to a schedule, helping reduce unnecessary operating time and save costs.

* Assigned **Tags** to AWS resources and created **Resource Groups** to manage resources by project, environment, or department, thereby improving resource search, tracking, and system administration.

* Understood the role of Tags in supporting resource management, cost control, operational automation, and effective organization of AWS infrastructure.

* Completed all hands-on exercises on AWS Security Hub, AWS Lambda, and Resource Groups, while also cleaning up all resources after finishing the Workshop to avoid incurring costs on the AWS account.
