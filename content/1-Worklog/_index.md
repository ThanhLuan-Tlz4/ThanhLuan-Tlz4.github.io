Throughout the **12 weeks** of participating in the **AWS First Cloud Journey** program, I gradually learned and practiced the core services of Amazon Web Services through theoretical lessons, hands-on Workshops, and the final project. The learning process not only helped me build a solid foundation in Cloud Computing but also gave me the opportunity to directly apply AWS services to develop a real-world system based on a Serverless architecture.

Through the Workshops and Capstone Project, I practiced deploying various AWS services such as Amazon EC2, Amazon VPC, Amazon S3, Amazon DynamoDB, AWS Lambda, Amazon API Gateway, Amazon Cognito, AWS Backup, AWS Storage Gateway, Amazon FSx, AWS Security Hub, AWS KMS, IAM, CloudWatch, and many other services. At the same time, I also learned the principles of Cloud architecture design based on the AWS Well-Architected Framework, including cost optimization, security, and system scalability.

During the final phase of the program, the team successfully developed a Serverless project on AWS, which helped me clearly understand the complete process from requirements analysis, architecture design, infrastructure deployment, Backend development, testing, and system optimization to product completion.

### Summary of the implementation weeks

**Week 1:** [Getting Started with AWS and Basic Services](1.1-week1/)

* Become familiar with the AWS First Cloud Journey program.
* Create an AWS account and learn about the AWS Management Console.
* Install AWS CLI and configure the working environment.
* Learn about the basic AWS service categories.

**Week 2:** [IAM and Access Management](1.2-week2/)

* Learn about AWS IAM.
* Manage Users, Groups, Roles, and Policies.
* Practice access control based on the Least Privilege principle.

**Week 3:** [Amazon VPC and Networking](1.3-week3/)

* Build a network system on AWS.
* Practice VPC, Subnet, Route Table, and Internet Gateway.
* Learn about Security Group and Network ACL.

**Week 4:** [Infrastructure as Code with AWS CloudFormation](1.4-week4/)

* Learn about Infrastructure as Code.
* Practice building and deploying CloudFormation Templates.
* Manage infrastructure using Stacks.

**Week 5:** [AWS Storage and Backup](1.5-week5/)

* Practice Amazon S3.
* Deploy AWS Backup.
* Configure AWS Storage Gateway.
* Understand the Hybrid Storage model.

**Week 6:** [Storage Services and Migration](1.6-week6/)

* Study the theory of Amazon S3, Storage Gateway, Snow Family, and Disaster Recovery.
* Deploy AWS Backup.
* Practice VM Import/Export.
* Participate in the AWS Event Meeting and exchange learning experiences.

**Week 7:** [Amazon FSx and Amazon S3](1.7-week7/)

* Review AWS storage services.
* Deploy Amazon FSx for Windows File Server.
* Practice managing data on Amazon S3.
* Compare storage solutions on AWS.

**Week 8:** [AWS Security and Cost Optimization](1.8-week8/)

* Learn about IAM, Cognito, Organizations, and AWS KMS.
* Practice AWS Security Hub.
* Optimize Amazon EC2 costs using AWS Lambda.
* Manage resources using Tags and Resource Groups.

**Week 9:** [Advanced IAM and AWS KMS](1.9-week9/)

* Manage EC2 access using Resource Tags.
* Practice IAM Permission Boundaries.
* Encrypt data using AWS KMS.
* Apply the Least Privilege principle in access control.

**Week 10:** [Capstone Project Preparation](1.10-week10/)

* Determine the final project topic.
* Analyze system requirements.
* Design the overall AWS architecture.
* Assign tasks to team members.

**Week 11:** [Serverless Backend Deployment](1.11-week11/)

* Build the Backend infrastructure on AWS.
* Deploy Amazon S3, DynamoDB, Lambda, and API Gateway.
* Build REST APIs and implement the Upload mechanism using Presigned URLs.
* Integrate Cognito, GuardDuty, Bedrock, and CloudWatch.

**Week 12:** [System Testing and Completion](1.12-week12/)

* Test the entire system.
* Fix errors and optimize the Backend.
* Test the integration of AWS services.
* Complete the documentation and prepare the project report.
